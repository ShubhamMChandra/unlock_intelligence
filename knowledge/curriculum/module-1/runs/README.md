# Runs: historical log of Module 1 instances

Every time Module 1 is taught live, it gets logged here. One file per cohort.

## Naming convention

`YYYY-MM-DD-cohort-descriptor.md`

The cohort descriptor should capture the most useful identifier for that run — the client name, the audience type, the program variant, whatever makes it easy to find later. Lowercase, hyphens, no spaces.

Examples:

- `2026-04-12-beta-cross-industry.md` (the first live run)
- `2026-05-17-acme-corp-internal.md` (a hypothetical future run for a specific client)
- `2026-06-02-accountants-private-cohort.md` (a hypothetical future audience-specific run)

## What each file should include

- **Pre-run context.** Who's in the room, how they got there, what you know about their expectations, any pre-work you sent, any overrides to the canonical run-of-show.
- **The plan as executed.** If it's materially the same as the canonical, reference it and describe only the deltas. If it's materially different, include the full run sheet. Lean toward including more context early on while the canonical is still young.
- **Cohort-specific decisions.** Volunteer pick, demo examples, starter Project specifics, anything tailored to this particular group.
- **What happened.** After the run, a retro: what landed, what didn't, where the instructor (you) wanted to change course live, specific moments worth remembering.
- **Feedback captured.** Quotes from attendees, survey results, testimonials you're allowed to keep.
- **What to promote to canonical.** At the end of the retro, explicitly identify what from this run should be pulled up into `../run-of-show.md` as a lasting change, and what stays cohort-specific.

## Why it matters

The learnings folder at `knowledge/learnings/` captures specific insights at specific dates. Those are cross-cutting and not tied to any one artifact. The runs folder is different — it captures the full context of each live instance of Module 1 so that future variants of the module can be grounded in what actually worked rather than in what we thought would work.

Over time, the runs folder is the primary evidence base for any claim about how Module 1 teaches. Anyone proposing a change to the canonical run-of-show should be able to point at runs to justify it.
